# Context Optimize 命令

专项分析 CLAUDE.md 文件的 Token 效率，识别冗余内容并提供精简建议。

## 执行流程

### Step 1: 收集 CLAUDE.md 文件

使用 Glob 扫描所有 CLAUDE.md 文件：

```
**/CLAUDE.md
```

### Step 2: 读取并分析每个文件

对每个文件执行：

1. **读取完整内容**
2. **统计基础指标**
   - 总行数
   - 字符数
   - 估算 Token 数

3. **结构分析**
   - 章节数量
   - 每章节行数
   - 代码块占比
   - 表格占比

### Step 3: Token 估算

使用以下公式估算 Token：

```
英文/代码: 字符数 / 4
中文: 字符数 / 2
混合内容: 字符数 / 3 (保守估计)
```

### Step 4: 内容分析

识别以下问题：

#### 冗余检测

1. **跨文件重复**
   - 子目录复制父级内容
   - 多个文件相同段落

2. **内部冗余**
   - 重复的说明文字
   - 过度的示例代码

3. **低频内容**
   - 详细的操作步骤 (应移到 docs/)
   - 完整的 API 文档 (应移到 Memory)
   - 历史变更记录

#### 效率问题

1. **列表 vs 表格**
   - 检测可以转换为表格的列表

2. **详细 vs 引用**
   - 检测可以用引用替代的详细说明

3. **冗长描述**
   - 检测可以精简的段落

## 优化技术参考

```markdown

- 训练任务的英文是 Training Job
- 训练任务对应的类是 TrainingJob
- 训练任务的数据库表是 training_jobs
- 训练任务的 API 路径是 /training-jobs

| 中文 | 英文 | 类 | 表 | API |
|------|------|-----|-----|-----|
```

