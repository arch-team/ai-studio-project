```python
MAX_PREEMPTION_COUNT = 3

async def _handle_preemption(self, job: TrainingJob) -> None:
    """处理抢占状态转换"""
    job.preemption_count += 1

    if job.preemption_count >= MAX_PREEMPTION_COUNT:
        job.fail(
            error_message="连续抢占次数超限",
            failure_reason="PreemptionExhausted",
        )
        await self._send_preemption_exhausted_alert(job)
    else:
        job.transition_to(JobStatus.PREEMPTED)
```

### 7.3 停滞检测算法 (T037c)

```python
def _calculate_change_rate(self, values: list[float]) -> float:
    """计算指标变化率"""
    if len(values) < 2:
        return 1.0  # 数据不足，返回高变化率（不判定为停滞）

    first_value = values[0]
    last_value = values[-1]

    if first_value == 0:
        return abs(last_value) if last_value != 0 else 0.0

    return abs((last_value - first_value) / first_value)
```

## 8. 风险和缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| HyperPod API 响应延迟 | 同步任务超时 | 配置合理的超时和重试策略 |
| 指标服务不可用 | 停滞检测失效 | 降级策略：跳过检测并记录日志 |
| 并发同步冲突 | 数据不一致 | 使用数据库行锁或乐观锁 |
| 告警风暴 | 通知过多 | 配置告警去重和冷却期 |
