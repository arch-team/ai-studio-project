## 六、执行计划

### 阶段 1: 自动化测试验证

```bash
# 1. 运行单元测试
cd backend
pytest tests/unit/modules/datasets -v --tb=short

# 2. 运行集成测试
pytest tests/integration/datasets -v --tb=short

# 3. 运行架构合规测试
pytest tests/architecture -v -k "dataset"

# 4. 检查测试覆盖率
pytest tests/unit/modules/datasets --cov=src/modules/datasets --cov-report=term-missing
```

### 阶段 2: 手动 API 验证

使用 httpie 或 curl 验证关键端点：

```bash
# 创建数据集
http POST localhost:8000/api/v1/datasets \
  Authorization:"Bearer $TOKEN" \
  name=test-dataset storage_type=s3 \
  storage_uri=s3://bucket/path dataset_type=image

# 列表查询
http GET localhost:8000/api/v1/datasets \
  Authorization:"Bearer $TOKEN" \
  dataset_type==image page==1 page_size==10

# 获取详情
http GET localhost:8000/api/v1/datasets/1 \
  Authorization:"Bearer $TOKEN"

# 更新
http PATCH localhost:8000/api/v1/datasets/1 \
  Authorization:"Bearer $TOKEN" \
  description="Updated" visibility=public

# 创建版本
http POST localhost:8000/api/v1/datasets/1/versions \
  Authorization:"Bearer $TOKEN" \
  version=v2

# 删除
http DELETE localhost:8000/api/v1/datasets/1 \
  Authorization:"Bearer $TOKEN"
```
