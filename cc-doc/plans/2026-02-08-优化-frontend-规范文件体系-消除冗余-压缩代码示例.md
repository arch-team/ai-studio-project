# 优化 frontend 规范文件体系：消除冗余，压缩代码示例

## Context

frontend/.claude/ 下的 13 个规范文件总计约 2,554 行 (~79KB)，预估每次会话消耗 40,000-60,000 tokens 的上下文。通过分析发现：
1. 存在 2 处高严重度的"双源维护"重叠（违反单一真实源原则）
2. 4 个大文件占总量 56%，其中包含大量完整代码实现（而非最小示例）
3. 职责边界整体清晰，不需要合并或拆分文件

**目标**: 消除重叠 + 压缩代码示例，预期减少 ~20% token 开销（2,554 → ~2,050 行），同时保持所有规范的指导价值。

## 验证方法

1. **单一真实源检查**: 确认技术栈版本只在 tech-stack.md 中定义，模块列表只在 project-config.md 中定义
2. **引用完整性**: 确认所有 `→ xxx.md §N` 引用指向实际存在的章节
3. **导航完整性**: 确认 CLAUDE.md 文档导航表列出所有 11 个 rules/ 文件
4. **速查卡片保留**: 确认每个文件的 §0 速查卡片完整保留
5. **内容无损**: 确认所有架构规则、约束条件、禁止项完整保留（只压缩了代码示例，未删除任何规则）
