# spec.md Token 优化计划

## 背景理解

### spec.md 在 Spec-Kit 中的作用
- **职责**: 功能规范文档（WHAT/WHY）
- **内容**: 用户故事、验收标准、术语定义、功能需求、成功标准
- **工作流位置**: `constitution → specify → plan → tasks → implement`
- **消费者**: 主要被 Claude Code 在生成 plan.md 和 tasks.md 时读取

### 当前状态
- **行数**: ~1342 行
- **Token 数**: ~34,823 tokens
- **主要章节**:
  1. Clarifications（历史问答）
  2. Terminology Standards（术语标准）
  3. User Scenarios & Testing（用户场景）
  4. Training Job State Model（状态模型）
  5. Requirements（功能需求 FR/NFR）
  6. API Design Standards（API 设计标准）
  7. Success Criteria（成功标准）

## 优化原则

1. **保持文档含义不变** - 所有优化仅减少冗余，不删除必要信息
2. **保持 Spec-Kit 功能完整** - 作为规范文档的核心功能必须保留
3. **优先删除对 Claude 理解无帮助的内容** - 如重复说明、冗长示例
4. **保留关键术语定义和验收标准** - 这些是 plan.md/tasks.md 生成的核心依据
