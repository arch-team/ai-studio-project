# Backend 异常处理框架统一化方案

## 1. 现状分析

### 1.1 异常定义分布

项目共有 **9 个异常定义文件**，约 **50 个自定义异常类**：

| 位置 | 异常数 | 职责 |
|------|--------|------|
| `src/shared/domain/exceptions.py` | 6 | DomainError 体系（业务规则异常） |
| `src/shared/infrastructure/security/exceptions.py` | 14 | SecurityError 体系（认证授权异常） |
| `src/modules/auth/domain/exceptions.py` | 14 | AuthError 体系（⚠️ 与 Security 重复） |
| `src/modules/training/domain/exceptions.py` | 12 | TrainingError + HyperPod 异常 |
| `src/modules/quotas/domain/exceptions.py` | 4 | QuotaError 体系 |
| `src/modules/spaces/domain/exceptions.py` | 5 | SpaceError 体系 |
| `src/modules/models/domain/exceptions.py` | 4 | ModelError 体系 |
| `src/modules/audit/domain/exceptions.py` | 2 | AuditLogError 体系 |

### 1.2 核心问题

#### 问题 1: Auth 与 Security 异常重复定义（🔴 严重）

**11 个异常**在两处重复定义：

```python
# shared/infrastructure/security/exceptions.py
class UserNotFoundError(SecurityError):
    def __init__(self, user_id: str):
        self.user_id = user_id
        super().__init__(f"User with id '{user_id}' not found", code="USER_NOT_FOUND")

# modules/auth/domain/exceptions.py
class UserNotFoundError(AuthError):
    def __init__(self, identifier: str | int):
        self.identifier = identifier
        super().__init__(f"User not found: {identifier}", code="USER_NOT_FOUND")
```

**影响**：
- exception_handlers.py 需要维护两套映射（行 66-80 和 行 82-93）
- 代码中通过别名区分（如 `AuthUserNotFoundError`），增加复杂度
- 维护负担增加，容易遗漏

#### 问题 2: 未映射异常（🟡 中等）

**21 个异常**未在 `DOMAIN_EXCEPTION_MAP` 中显式映射：

| 模块 | 未映射异常 |
|------|-----------|
| training | `TrainingJobNotFoundError`, `InvalidJobStateError`, `DuplicateJobNameError`, `CheckpointNotFoundError` |
| quotas | `QuotaNotFoundError`, `QuotaExceededError`, `DuplicateConfigError` |
| spaces | `SpaceNotFoundError`, `InvalidSpaceStateError`, `SpaceQuotaExceededError` |
| models | `ModelNotFoundError`, `InvalidModelStateError`, `DuplicateModelVersionError` |
| audit | `AuditLogNotFoundError` |

**当前行为**：依赖继承链回退，最终默认 400 Bad Request（不理想）

#### 问题 3: 响应格式不一致（🟡 中等）

```python
# Domain 异常响应
{"detail": "...", "error_type": "TrainingJobNotFoundError"}

# Security 异常响应
{"detail": "...", "error_code": "INVALID_CREDENTIALS"}
```

**问题**：字段名不统一（`error_type` vs `error_code`），前端解析困难

#### 问题 4: 不符合 RFC 7807 标准（🟢 改进项）

当前响应缺少：
- `type`: 问题类型 URI
- `title`: 人类可读的简短标题
- `status`: HTTP 状态码（冗余但标准要求）
- `instance`: 问题发生位置

## 2. SDK 和框架调研

| 库 | RFC 兼容 | FastAPI 集成 | 维护状态 | 推荐度 |
|----|----------|-------------|---------|--------|
| **fastapi-problem** | RFC 9457 ✅ | 原生 ✅ | 活跃（2026-01） | ⭐⭐⭐⭐⭐ |
| python-httpproblem | RFC 7807 ✅ | 手动 | 不维护（2020） | ⭐⭐ |

| 自建方案 | 可选 | 自定义 | - | ⭐⭐⭐ |

### 2.2 fastapi-problem 优势

```python
from fastapi_problem import Problem

class NotFoundProblem(Problem):
    type = "https://api.example.com/problems/not-found"
    title = "Resource Not Found"
    status = 404

# 使用
raise NotFoundProblem(
    detail="Training job 123 not found",
    extensions={"job_id": 123, "error_code": "JOB_NOT_FOUND"}
)
```

**输出格式（RFC 9457 兼容）**：
```json
{
  "type": "https://api.example.com/problems/not-found",
  "title": "Resource Not Found",
  "status": 404,
  "detail": "Training job 123 not found",
  "job_id": 123,
  "error_code": "JOB_NOT_FOUND"
}
```

## 3. 推荐方案：混合架构

### 3.1 设计原则

1. **保留 DDD 异常体系**：业务层继续抛出 `DomainError`、`SecurityError` 等
2. **引入 fastapi-problem**：在 API 层将 DDD 异常转换为 RFC 9457 响应
3. **消除重复定义**：统一 Auth 和 Security 异常
4. **补全映射**：所有模块异常显式映射

### 3.2 架构图


### 3.3 关键改造点

#### 改造 1: 统一响应格式

**改造文件**: `src/shared/api/exception_handlers.py`

```python
from fastapi_problem import Problem

# 统一 Problem 响应格式
def create_problem_response(
    exc: Exception,
    status_code: int,
    request: Request,
) -> JSONResponse:
    """将异常转换为 RFC 9457 Problem Details 响应"""
    error_type = type(exc).__name__
    error_code = getattr(exc, "code", error_type.upper())

    content = {
        "type": f"https://api.ai-studio.example/problems/{error_type}",
        "title": _get_title(exc),
        "status": status_code,
        "detail": exc.message if hasattr(exc, "message") else str(exc),
        "instance": str(request.url.path),
        "error_code": error_code,  # 统一字段名
    }

    # 附加异常特有属性
    for attr in ["entity_type", "entity_id", "locked_until", "violations"]:
        if hasattr(exc, attr):
            content[attr] = getattr(exc, attr)

    return JSONResponse(status_code=status_code, content=content)
```

#### 改造 2: 消除重复异常

**策略**：删除 `modules/auth/domain/exceptions.py` 中的重复定义，统一使用 `shared/infrastructure/security/exceptions.py`

**保留**：`AccountInactiveError`（Auth 模块独有）

#### 改造 3: 补全异常映射


## 5. 确定的技术决策

### 决策 1: SDK 选择 ✅

**决定**: 引入 `fastapi-problem` 库

- 符合 RFC 9457 标准
- 社区活跃维护（2026-01 更新）
- 开箱即用，减少重复造轮子

### 决策 2: 异常合并策略 ✅

**决定**: 重导出方式（保持向后兼容）

```python
# modules/auth/domain/exceptions.py (改造后)
"""Auth domain exceptions - 重导出 shared/security 异常以保持向后兼容."""

from src.shared.infrastructure.security.exceptions import (
    SecurityError,
    UserNotFoundError,
    InvalidCredentialsError,
    AccountLockedError,
    TokenExpiredError,
    InvalidTokenError,
    PasswordExpiredError,
    PasswordTooWeakError,
    PasswordHistoryViolationError,
    InsufficientPermissionsError,
    SSOError,
    SSODegradedModeError,
)

# Auth 模块独有异常
class AuthError(SecurityError):
    """Base exception for authentication errors."""
    def __init__(self, message: str, code: str = "AUTH_ERROR"):
        super().__init__(message, code=code)

class AccountInactiveError(AuthError):
    """Raised when account is not active."""
    def __init__(self, message: str = "Account is not active"):
        super().__init__(message, code="ACCOUNT_INACTIVE")

class TokenError(AuthError):
    """Base exception for token errors."""
    def __init__(self, message: str = "Token error", code: str = "TOKEN_ERROR"):
        super().__init__(message, code=code)

__all__ = [
    # 重导出的异常
    "SecurityError",
    "UserNotFoundError",
    "InvalidCredentialsError",
    "AccountLockedError",
    "TokenExpiredError",
    "InvalidTokenError",
    "PasswordExpiredError",
    "PasswordTooWeakError",
    "PasswordHistoryViolationError",
    "InsufficientPermissionsError",
    "SSOError",
    "SSODegradedModeError",
    # Auth 模块独有
    "AuthError",
    "AccountInactiveError",
    "TokenError",
]
```

### 决策 3: 错误类型 URI 格式 ✅

**决定**: 使用 error_code 格式

```
https://api.ai-studio.example/errors/ENTITY_NOT_FOUND
https://api.ai-studio.example/errors/INVALID_CREDENTIALS
https://api.ai-studio.example/errors/QUOTA_EXCEEDED
```

**优势**:
- 与现有 `error_code` 字段保持一致
- 可直接映射到 API 文档
- 比类名更稳定（重构不影响）
