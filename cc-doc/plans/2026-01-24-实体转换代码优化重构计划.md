> **目标**: 减少后端项目中 `_to_entity`/`_to_model`/`_map_entity_fields` 等转换代码的冗余，提升可维护性

## 1. 问题分析

| `_to_entity` | 11 | ~200 行 | 手动字段赋值 |

## 2. 优化方案

### 2.1 方案一：增强 EntitySchema（Schema 层优化）

**核心思想**: 自动映射同名字段，只需声明枚举映射规则

```python
# shared/api/schemas/enhanced_base.py
class AutoMappingEntitySchema(EntitySchema[Entity]):
    """自动映射同名字段的 EntitySchema"""

    # 子类配置
    _enum_mappings: ClassVar[dict[str, type[Enum]]] = {}  # 枚举映射
    _custom_mappings: ClassVar[dict[str, Callable]] = {}  # 自定义转换
    _exclude_fields: ClassVar[set[str]] = set()  # 排除字段

    @classmethod
    def from_entity(cls, entity: Entity, **extra) -> Self:
        fields = cls._auto_map(entity)
        fields.update(extra)
        return cls(**fields)

    @classmethod
    def _auto_map(cls, entity: Entity) -> dict:
        result = {}
        for field_name in cls.model_fields:
            if field_name in cls._exclude_fields:
                continue
            if not hasattr(entity, field_name):
                continue

            value = getattr(entity, field_name)

            # 枚举自动转换
            if field_name in cls._enum_mappings:
                value = EnumMapper.to_api(value, cls._enum_mappings[field_name])
            # 自定义转换
            elif field_name in cls._custom_mappings:
                value = cls._custom_mappings[field_name](entity)

            result[field_name] = value
        return result
```

**使用示例**:
```python
# 迁移前: 14 行映射代码
class TrainingJobSummary(EntitySchema["TrainingJob"]):
    @classmethod
    def _map_entity_fields(cls, entity):
        return {"id": entity.id, "job_name": entity.job_name, ...}  # 14 行

# 迁移后: 3 行配置
class TrainingJobSummary(AutoMappingEntitySchema["TrainingJob"]):
    id: int
    job_name: str
    status: JobStatusEnum
    priority: JobPriorityEnum
    # ... 字段声明

    _enum_mappings = {"status": JobStatusEnum, "priority": JobPriorityEnum}

# Detail 继承 Summary，零重复
class TrainingJobDetail(TrainingJobSummary):
    description: str | None
    # ... 新增字段

    _enum_mappings = {
        **TrainingJobSummary._enum_mappings,
        "distribution_strategy": DistributionStrategyEnum,
    }
```

**预期收益**: Schema 层代码减少 **60-70%**

### 2.2 方案二：声明式字段映射（仓库层优化）

**核心思想**: 使用字段映射配置替代手动赋值

```python
# shared/infrastructure/field_mapping.py
@dataclass(frozen=True)
class FieldMapping:
    """字段映射配置"""
    entity_field: str
    model_field: str | None = None  # 默认同名
    to_entity: Callable | None = None
    to_model: Callable | None = None
    skip_update: bool = False  # update 时跳过


class DeclarativeMappingMixin:
    """声明式映射混入类"""

    _field_mappings: ClassVar[list[FieldMapping]] = []
    _entity_class: ClassVar[type] = None
    _skip_on_update: ClassVar[set[str]] = {"id", "created_at"}

    def _to_entity(self, model: ModelT) -> EntityT:
        kwargs = {}
        for m in self._field_mappings:
            model_field = m.model_field or m.entity_field
            value = getattr(model, model_field, None)
            if m.to_entity:
                value = m.to_entity(value)
            kwargs[m.entity_field] = value
        return self._entity_class(**kwargs)

    def _to_model(self, entity: EntityT) -> ModelT:
        kwargs = {}
        for m in self._field_mappings:
            if m.entity_field in self._skip_on_update:
                continue
            model_field = m.model_field or m.entity_field
            value = getattr(entity, m.entity_field, None)
            if m.to_model:
                value = m.to_model(value)
            kwargs[model_field] = value
        return self._model_class(**kwargs)

    def _update_model(self, model: ModelT, entity: EntityT) -> None:
        for m in self._field_mappings:
            if m.skip_update or m.entity_field in self._skip_on_update:
                continue
            model_field = m.model_field or m.entity_field
            value = getattr(entity, m.entity_field, None)
            if m.to_model:
                value = m.to_model(value)
            setattr(model, model_field, value)
```

**使用示例**:
```python
# 迁移前: 88 行转换代码
class TrainingJobRepository(EnhancedBaseRepository):
    def _to_entity(self, model): ...  # 48 行
    def _to_model(self, entity): ...  # 24 行
    def _update_model(self, model, entity): ...  # 16 行

# 迁移后: ~20 行配置
class TrainingJobRepository(
    DeclarativeMappingMixin,
    EnhancedBaseRepository[TrainingJob, TrainingJobModel, int],
):
    _entity_class = TrainingJob
    _field_mappings = [
        FieldMapping("id"),
        FieldMapping("job_name"),
        FieldMapping("status", to_entity=lambda v: JobStatus(v.value)),
        FieldMapping("priority", to_entity=lambda v: JobPriority(v.value)),
        FieldMapping("learning_rate", to_entity=float, to_model=Decimal),
        # ... 字段配置
    ]
    _skip_on_update = {"id", "created_at", "job_name"}  # job_name 不可变
```

**预期收益**: 仓库层代码减少 **50-60%**

### 2.3 方案三：增强 EnumMapper

**核心思想**: 统一所有枚举转换调用

```python
# shared/utils/mapping.py (增强)
class EnumMapper:
    # 现有方法...

    @staticmethod
    def model_to_domain(
        model_enum: Enum | None,
        domain_class: type[DomainEnumT]
    ) -> DomainEnumT | None:
        """ORM Model → Domain Entity (相同值格式)"""
        return domain_class(model_enum.value) if model_enum else None

    @staticmethod
    def domain_to_model(
        domain_enum: DomainEnumT | None,
        model_class: type[Enum]
    ) -> Enum | None:
        """Domain Entity → ORM Model (相同值格式)"""
        return model_class(domain_enum.value) if domain_enum else None
```

## 8. 备选方案

如果声明式方案过于复杂，可考虑：

1. **代码生成**: 使用脚本从 Entity/Model 定义生成转换代码
2. **第三方库**: 引入 `cattrs` 或 `pydantic-to-dataclass` 等
3. **最小化改进**: 仅统一枚举转换，不改变整体模式
