## 1. 详细审查结果

### 1.1 ⛔ 高风险 - 建议重构

#### 问题 1: `infrastructure/storage/s3_client.py` (184 行)

| 项目 | 分析 |
|------|------|
| **现状** | 使用同步 boto3 + `run_in_executor` 包装为异步 |
| **问题** | 手动异步包装不够优雅，每个方法重复 `loop.run_in_executor` 模式 |
| **推荐方案** | 使用 **aioboto3** 库 (原生异步 boto3 封装) |
| **收益** | 减少 ~50 行重复代码，原生异步性能更好 |
| **风险等级** | ⛔ **HIGH** - 存在更成熟的替代方案 |

```python
# 当前实现 (重复模式)
async def upload_file(self, local_path: str, remote_path: str, ...) -> str:
    loop = asyncio.get_event_loop()
    def _upload() -> str:
        self._s3_client.upload_file(...)
    return await loop.run_in_executor(None, _upload)

# 推荐方案 (aioboto3)
async def upload_file(self, local_path: str, remote_path: str, ...) -> str:
    async with self._session.client("s3") as s3:
        await s3.upload_file(...)
```

**安装**: `pip install aioboto3`

---

#### 问题 2: `infrastructure/query_builder.py` (122 行)

| 项目 | 分析 |
|------|------|
| **现状** | 自定义 QueryBuilder 类，处理分页、排序、软删除过滤 |
| **问题** | 功能可由 `fastapi-pagination` 或 SQLAlchemy 原生 API 替代 |
| **推荐方案** | 使用 **fastapi-pagination** 库 |
| **收益** | 减少 ~100 行代码，获得社区维护的成熟方案 |
| **风险等级** | ⛔ **HIGH** - 社区有更完善的方案 |

**注意**: 该组件包含枚举自动转换逻辑 (`_convert_enum_value`)，这是项目特定需求，可保留为独立工具函数。

```python
# 推荐方案
from fastapi_pagination import Page, paginate
from fastapi_pagination.ext.sqlalchemy import paginate as sqlalchemy_paginate

@router.get("/jobs", response_model=Page[TrainingJobResponse])
async def list_jobs(db: AsyncSession = Depends(get_db)):
    return await sqlalchemy_paginate(db, select(TrainingJobModel))
```

**安装**: `pip install fastapi-pagination`
