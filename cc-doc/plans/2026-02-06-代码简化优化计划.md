# 代码简化优化计划

## Context

项目已经过一轮架构优化（Mixin 模式重构），代码质量良好。但仍存在明显的重复代码和不一致模式：

- **前端**：10 个功能模块各自重复实现 `getAuthHeaders()`、`buildUrl()`、`handleResponse()`，而已有的 `shared/api/client.ts`（ApiClient）未被使用
- **后端**：`ResourceQuotaService` 等仍手写 CRUD 逻辑（157 行），而 BaseApplicationService Mixin 可将其简化到 ~40 行
- **前端**：`TrainingStatusBadge` 和 `ModelStatusBadge` 结构完全相同，仅数据映射不同

**预期收益**：减少 ~800-1000 行重复代码，统一代码模式，降低维护成本。

---

## Phase 1: 前端 API 客户端统一

**目标**：消除各模块重复的 `getAuthHeaders/buildUrl/handleResponse`，统一使用 `shared/api/client.ts`

### Step 1.1: 增强 ApiClient 自动认证

修改 `frontend/src/shared/api/client.ts`：
- 在 `request()` 方法中自动从 `localStorage` 读取 `access_token` 并注入 Authorization header
- 增强 `buildQueryString()` 支持数组参数（当前只处理简单值，各模块的 `buildUrl` 支持数组 `append`）
- 保持向后兼容：`setAuthToken/clearAuthToken` 仍可用于手动覆盖

```typescript
// request() 方法中自动注入 token
const token = localStorage.getItem('access_token');
if (token && !headers['Authorization']) {
  headers['Authorization'] = `Bearer ${token}`;
}
```

### Step 1.2: 逐模块迁移 API 函数

将以下 **9 个 API 文件** 从直接 `fetch` 迁移到 `apiClient`：

| 文件 | 当前行数 | 迁移后预估 |
|------|---------|-----------|
| `features/training/api/trainingJobApi.ts` | 255 行 | ~170 行 |
| `features/models/api/modelApi.ts` | 201 行 | ~130 行 |

| `features/spaces/api/spaceApi.ts` | ~150 行 | ~100 行 |
| `features/admin/api/userApi.ts` | ~180 行 | ~120 行 |
| `features/audit/api/auditApi.ts` | ~100 行 | ~70 行 |
| `features/billing/api/billingApi.ts` | ~120 行 | ~80 行 |
| `features/monitoring/api/monitoringApi.ts` | ~150 行 | ~100 行 |
| `features/reports/api/reportsApi.ts` | ~120 行 | ~80 行 |
| `features/templates/api/templateApi.ts` | ~100 行 | ~70 行 |

**迁移模式**（以 trainingJobApi.ts 为例）：

```typescript
// 迁移前 (每个文件重复 ~60 行基础代码)
function getAuthHeaders(): HeadersInit { ... }
function buildUrl(path: string, params?: Record<string, unknown>): string { ... }
async function handleResponse<T>(response: Response): Promise<T> { ... }

export async function fetchTrainingJobs(filters: TrainingJobFilters = {}): Promise<TrainingJobListResponse> {
  const url = buildUrl('/training-jobs', { page: filters.page, ... });
  const response = await fetch(url, { method: 'GET', headers: getAuthHeaders() });
  return handleResponse<TrainingJobListResponse>(response);
}

// 迁移后 (删除 60 行重复代码，每个函数简化)
import { apiClient } from '@shared/api/client';

export async function fetchTrainingJobs(filters: TrainingJobFilters = {}): Promise<TrainingJobListResponse> {
  return apiClient.get<TrainingJobListResponse>('/training-jobs', {
    params: { page: filters.page, ... },
  });
}
```

**每个文件删除 3 个重复函数（~60 行），10 个文件共减少 ~600 行。**

## Phase 2: 前端通用组件提取

### Step 2.1: 提取通用 StatusBadge 组件

创建 `frontend/src/shared/components/StatusBadge.tsx`：

```typescript
import { StatusIndicator, StatusIndicatorProps } from '@cloudscape-design/components';

interface StatusBadgeProps<S extends string> {
  status: S;
  typeMap: Record<S, StatusIndicatorProps['type']>;
  labelMap: Record<S, string>;
}

export function StatusBadge<S extends string>({ status, typeMap, labelMap }: StatusBadgeProps<S>) {
  const type = typeMap[status] || 'info';
  const label = labelMap[status] || status;
  return <StatusIndicator type={type}>{label}</StatusIndicator>;
}
```

简化 `TrainingStatusBadge.tsx` 和 `ModelStatusBadge.tsx` 为薄包装层：

function createEntityKeys<T extends string>(entity: T) {
  return {
    all: [entity] as const,
    lists: () => [entity, 'list'] as const,
    list: (filters: Record<string, unknown>) => [entity, 'list', filters] as const,
    details: () => [entity, 'detail'] as const,
    detail: (id: string) => [entity, 'detail', id] as const,
  };
}

export const queryKeys = {

## Phase 3: 后端 Service 层统一

### Step 3.1: 迁移 ResourceQuotaService 到 BaseApplicationService

**文件**：`backend/src/modules/quotas/application/services/resource_quota_service.py`

当前 157 行手动 CRUD → 迁移后 ~40 行：

```python
# 迁移后
class ResourceQuotaService(BaseApplicationService[ResourceQuota, int]):
    _not_found_error_factory = lambda id: QuotaNotFoundError(identifier=id)

    def __init__(self, repository: IResourceQuotaRepository) -> None:
        super().__init__(repository, "ResourceQuota")

    async def create_quota(self, data: dict[str, Any]) -> ResourceQuota:
        """创建配额。Raises: DuplicateQuotaNameError"""
        await self._validate_unique_field("name", data["name"], DuplicateQuotaNameError)
        now = utc_now()
        quota = ResourceQuota(id=0, name=data["name"], ..., created_at=now, updated_at=now)
        return await self._repository.create(quota)

    async def update_quota(self, quota_id: int, data: dict[str, Any]) -> ResourceQuota:
        """更新配额。Raises: QuotaNotFoundError, DuplicateQuotaNameError"""
        quota = await self._get_or_raise(quota_id)
        # 名称唯一性检查
        new_name = data.get("name")
        if new_name and new_name != quota.name:
            await self._validate_unique_field("name", new_name, DuplicateQuotaNameError)
        # 更新字段由 repository._updatable_fields 控制
        return await self._update_entity(quota_id, data)
```

**减少 ~117 行**：`get_quota()` → `_get_or_raise()`，`delete_quota()` → `_delete_entity()`，`list_quotas()` → `_list_entities()`。

### Step 3.2: 审查其他未迁移的 CRUD Service

以下服务**不需要迁移**（它们是专用服务，不遵循标准 CRUD 模式）：

| 服务 | 原因 |
|------|------|
| `HyperPodService` | 外部 AWS 服务集成，无标准 CRUD |
| `ReportService` | 报表生成逻辑，非实体 CRUD |
| `PricingModelService` | 计算逻辑，非实体 CRUD |
| `StallDetectionService` | 检测逻辑 |
| `AuthService/RBACService/PasswordService` | 认证专用逻辑 |
| `DatasetUploadService/FsxSyncService` | 上传/同步专用 |
| `ClusterHealthService/PrometheusService` | 监控专用 |
| `MLflowService` | 外部服务集成 |

**可选迁移候选**：

| 服务 | 当前行数 | 分析 |
|------|---------|------|
| `AuditService` | ~50 行 | 主要是写入和列表查询，可部分受益 |
| `UserService` | ~40 行 | auth 模块特殊逻辑，迁移收益低 |

**结论**：仅迁移 `ResourceQuotaService`，其他服务保持现状。收益/风险比最优。
