### 2. 危险操作确认机制 (新增章节)

**位置**: API Design Standards 新增章节

**补充内容**:
```markdown
### 危险操作确认机制

**需要二次确认的操作**:
- 删除训练任务（及其检查点）
- 删除数据集或数据集版本
- 终止运行中的训练任务
- 删除用户账号
- 清空资源配额

**确认方式**:
- **前端**: 弹窗确认，显示操作影响范围
- **API**: 请求需包含 `confirm=true` 参数，否则返回 HTTP 428 Precondition Required
- **响应格式** (缺少 confirm 参数时):
  ```json
  {
    "error": {
      "code": "CONFIRMATION_REQUIRED",
      "message": "此操作不可逆，请确认后重试",
      "details": {
        "affected_resources": ["checkpoint-1", "checkpoint-2"],
        "confirm_param": "confirm=true"
      }
    }
  }
  ```
```

### 4. 通知机制 (新增 FR-027)

**位置**: Functional Requirements

**补充内容**:
```markdown
- **FR-027**: 系统必须提供完整的通知机制，支持多渠道通知和用户订阅配置。
  - **通知渠道**:
    - 邮件通知（必选）: 用于重要事件（任务完成/失败、预算预警）
    - 站内信（必选）: 平台内通知中心，支持已读/未读状态
    - Webhook（可选）: 支持用户配置外部系统回调 URL
  - **通知类型**:
    | 通知类型 | 触发场景 | 默认渠道 |
    |---------|---------|---------|
    | 任务状态变更 | 训练完成/失败/被抢占 | 站内信 + 邮件 |
    | 预算预警 | 使用率达到 80%/90%/100% | 邮件 |
    | 存储容量告警 | 使用率达到 80%/90% | 邮件 |
    | 训练停滞告警 | 30 分钟无进展 | 站内信 + 邮件 |
    | Space 状态变更 | 超时暂停/被抢占 | 站内信 |
  - **订阅配置**: 用户可在个人设置中配置接收哪些类型的通知、通过哪个渠道
  - **通知模板**: 每种通知类型有标准模板，包含关键信息和操作链接
  - 🔧 **实施约束**: 邮件通过 Amazon SES 发送，站内信存储在 `notifications` 表
```

### 5. 数据集生命周期管理 (补充 FR-005/FR-006)

**位置**: FR-005 和 FR-006

**补充内容**:
```markdown
**数据集删除策略** (补充到 FR-005):
- 有训练任务引用的数据集版本禁止删除，返回 HTTP 409 Conflict
- 支持软删除（标记为 `archived`），保留元数据供审计
- 硬删除需要 Project Owner 权限 + 二次确认（参见危险操作确认机制）
- 删除操作记录到审计日志

**数据集访问权限** (补充到 FR-006):
- 数据集默认项目级隔离，仅项目成员可访问
- 支持跨项目共享: Project Owner 可授权其他项目只读访问
- 共享记录存储在 `dataset_shares` 表，包含授权人、被授权项目、权限级别、过期时间
```

**位置**: FR-013

**补充内容**:
```markdown
**模型审批流程** (补充):
- **审批角色**: Project Owner 或 Platform Admin 可审批模型
- **审批操作**:
  - `POST /models/{id}/actions/approve`: 批准模型，状态变为 Approved
  - `POST /models/{id}/actions/reject`: 拒绝模型，状态变为 Rejected，需提供拒绝原因
- **审批记录**: 审批人、审批时间、审批意见记录到审计日志
- **拒绝后处理**: 用户可修改模型元数据后重新提交审批（状态从 Rejected 回到 Registered）
```

### 8. 训练任务工作流编排 (新增 FR-028)

**位置**: Functional Requirements

**补充内容**:
```markdown
- **FR-028**: 系统必须支持训练任务工作流编排，允许用户定义任务依赖关系形成 DAG（有向无环图）。
  - **依赖类型**:
    - `depends_on`: 前置任务完成（无论成功失败）后启动
    - `on_success`: 前置任务成功时触发
    - `on_failure`: 前置任务失败时触发（用于清理或告警任务）
  - **典型场景**:
    ```
    数据预处理 (on_success) → 模型训练 (on_success) → 模型评估 (on_success) → 模型注册
                                    ↓ (on_failure)
                              失败通知任务
    ```
  - **工作流状态**:
    - `Pending`: 等待启动
    - `Running`: 工作流执行中
    - `Completed`: 所有任务成功完成
    - `PartiallyFailed`: 部分任务失败，但工作流继续执行
    - `Failed`: 关键任务失败，工作流终止
  - **API 端点**:
    - `POST /workflows`: 创建工作流
    - `GET /workflows/{id}`: 查询工作流状态和任务执行情况
    - `POST /workflows/{id}/actions/cancel`: 取消工作流（终止所有运行中任务）
  - **资源配额**: 工作流内所有任务共享提交者的资源配额
  - 🔧 **实施约束**: 工作流定义存储在 `workflows` 表，任务依赖关系存储在 `workflow_tasks` 表
```

### 9. API 错误码列表 (记录现有实现)

**位置**: API Design Standards 新增章节

**补充内容**:
```markdown
### API 错误响应格式

系统采用统一的错误响应格式（基于 RFC 9457 简化版）:

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "人类可读的错误描述",
    "details": { "field": "value" },
    "trace_id": "req-xxx"
  }
}
```

**标准错误码列表**:

| 错误码 | HTTP 状态码 | 说明 |
|-------|------------|------|
| `ENTITY_NOT_FOUND` | 404 | 请求的资源不存在 |
| `VALIDATION_ERROR` | 422 | 参数验证失败 |
| `DUPLICATE_ENTITY` | 409 | 资源已存在（如重复的任务名称） |
| `INVALID_STATE_TRANSITION` | 409 | 非法状态转换（如已完成任务无法暂停） |
| `RESOURCE_QUOTA_EXCEEDED` | 429 | 资源配额不足 |
| `AUTHENTICATION_ERROR` | 401 | 认证失败 |
| `INVALID_CREDENTIALS` | 401 | 凭证无效 |
| `TOKEN_EXPIRED` | 401 | Token 过期 |
| `INSUFFICIENT_PERMISSIONS` | 403 | 权限不足 |
| `ACCOUNT_LOCKED` | 423 | 账户被锁定 |
| `CONFIRMATION_REQUIRED` | 428 | 危险操作需要确认 |
| `INTERNAL_ERROR` | 500 | 服务器内部错误 |

**实现位置**: `backend/src/shared/domain/problem.py`, `backend/src/shared/domain/exceptions.py`
```
