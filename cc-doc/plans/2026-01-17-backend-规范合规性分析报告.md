# Backend 规范合规性分析报告

> **分析日期**: 2026-01-17
> **范围**: `backend/src/modules/**` 与 CLAUDE.md / ARCHITECTURE.md 规范对比

---

## 概述

本次分析检查 backend 项目是否符合 `CLAUDE.md` 和 `docs/ARCHITECTURE.md` 中定义的架构规范。发现 **3 类违规问题**，共 **9 处具体违规**。

| 违规类别 | 严重性 | 数量 | 影响 |
|---------|--------|------|------|
| 模块导出违规 (M2/M3) | P1 高 | 1 处 | 模块封装泄露 |
| 跨模块依赖违规 (R3) | P0 严重 | 1 处 | 破坏架构边界 |
| 异常类型违规 (E1) | P1 高 | 7 处 | 错误处理不一致 |

---

## 1. 违规详情

### 1.1 模块导出违规 (M2/M3) - auth 模块

**规范要求** (ARCHITECTURE.md 5.3 节):
> 禁止导出 ORM 模型和仓库实现

**违规位置**: `backend/src/modules/auth/__init__.py:70-77`

```python
# ❌ 违规：导出了 ORM 模型
"UserModel",
"LoginAttemptModel",
"PasswordHistoryModel",

# ❌ 违规：导出了仓库实现
"UserRepositoryImpl",
"LoginAttemptRepositoryImpl",
"PasswordHistoryRepositoryImpl",
```

**对比其他模块** (符合规范):
- `training/__init__.py` - ✅ 只导出接口 `ITrainingJobRepository`
- `models/__init__.py` - ✅ 只导出接口 `IModelRepository`
- `quotas/__init__.py` - ✅ 只导出接口 `IResourceLimitConfigRepository`

---

### 1.2 跨模块依赖违规 (R3) - models 模块

**规范要求** (ARCHITECTURE.md 3.3 节):
> 禁止直接导入其他模块的 Repository 实现

**违规位置**: `backend/src/modules/models/api/dependencies.py:9`

```python
# ❌ 违规：直接导入 training 模块的仓库实现
from src.modules.training.infrastructure.repositories import TrainingJobRepository
```

**使用场景**: `get_model_service()` 注入 training 的仓库

**规范做法**: 应通过 `shared/domain/interfaces/` 定义接口，或使用事件通信

---

### 1.3 异常类型违规 (E1) - 多处仓库实现

**规范要求** (ARCHITECTURE.md 6.1 节):
> 必须使用 DomainError 体系，禁止 ValueError/Exception

**违规位置** (7 处):

| 文件 | 行号 | 代码 |
|------|------|------|
| `auth/infrastructure/repositories/user_repository_impl.py` | 105 | `raise ValueError(f"User with id {user.id} not found")` |
| `quotas/infrastructure/repositories/resource_limit_config_repository_impl.py` | 160 | `raise ValueError(f"ResourceLimitConfig with id {config.id} not found")` |
| `training/api/schemas/requests.py` | 70 | `raise ValueError(...)` |
| `training/infrastructure/repositories/checkpoint_repository_impl.py` | 94 | `raise ValueError(f"Checkpoint with id {checkpoint.id} not found")` |
| `training/infrastructure/repositories/training_job_repository_impl.py` | 195 | `raise ValueError(f"TrainingJob with id {job.id} not found")` |
| `spaces/infrastructure/repositories/space_repository_impl.py` | 127 | `raise ValueError(f"Space with id {space.id} not found")` |
| `models/infrastructure/repositories/model_repository_impl.py` | 206 | `raise ValueError(f"Model with id {model.id} not found")` |

## 2. 符合规范的部分

### 2.1 黄金法则 R1 - Domain 层隔离 ✅

所有模块的 Domain 层未发现跨模块导入。

### 2.2 黄金法则 R4 - Auth 依赖例外 ✅

其他模块仅在 API 层导入 auth 的认证依赖（如 `require_admin`）。

### 2.3 分层依赖方向 ✅

各模块的分层结构正确：API → Application → Domain ← Infrastructure

### 2.4 命名约定 ✅

文件命名符合规范：`{entity}_repository.py`, `{entity}_repository_impl.py`, `{entity}_model.py`

### 2.5 测试架构合规 ✅

`tests/unit/test_architecture_compliance.py` 已覆盖核心规则验证。

## 3. 修复方案

### 3.1 修复 auth/__init__.py 导出违规

**操作**: 移除 ORM 模型和仓库实现的导出

```python
# 移除以下导出
- "UserModel",
- "LoginAttemptModel",
- "PasswordHistoryModel",
- "UserRepositoryImpl",
- "LoginAttemptRepositoryImpl",
- "PasswordHistoryRepositoryImpl",
```

**关键文件**: `backend/src/modules/auth/__init__.py`

---

### 3.2 修复 models 模块跨模块依赖

**选项 A - 共享接口方案 (推荐)**:

1. 定义接口: `shared/domain/interfaces/training_job_reader.py`
```python
class ITrainingJobReader(ABC):
    async def get_by_id(self, job_id: int) -> Optional[TrainingJobInfo]: ...
```

2. 在 training 模块实现接口
3. 在 models/api/dependencies.py 依赖接口而非实现

**选项 B - 移除依赖**:

如果 ModelService 不需要 TrainingJobRepository，直接移除依赖。

**关键文件**:
- `backend/src/modules/models/api/dependencies.py`
- `backend/src/shared/domain/interfaces/` (新增)

---

### 3.3 修复 ValueError 异常违规

**操作**: 将所有 `ValueError` 替换为对应的 `EntityNotFoundError`

```python
# Before
raise ValueError(f"User with id {user.id} not found")

# After
from src.shared.domain import EntityNotFoundError
raise EntityNotFoundError(entity_type="User", entity_id=str(user.id))
```

**关键文件** (7 处):
- `backend/src/modules/auth/infrastructure/repositories/user_repository_impl.py:105`
- `backend/src/modules/quotas/infrastructure/repositories/resource_limit_config_repository_impl.py:160`
- `backend/src/modules/training/api/schemas/requests.py:70`
- `backend/src/modules/training/infrastructure/repositories/checkpoint_repository_impl.py:94`
- `backend/src/modules/training/infrastructure/repositories/training_job_repository_impl.py:195`
- `backend/src/modules/spaces/infrastructure/repositories/space_repository_impl.py:127`
- `backend/src/modules/models/infrastructure/repositories/model_repository_impl.py:206`
