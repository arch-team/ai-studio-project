# 后端代码清理计划 - 死代码与过程文件清理

## Context

最近三轮代码优化（净减 1,386 行）将多个大文件拆分为更小的模块，但留下了以下需要清理的问题：
1. 拆分后，旧的 `.py` 文件与新的同名包目录并存，旧文件成为死代码
2. 散落在源码目录中的 5 个 Claude 生成的 OPTIMIZATION_REPORT 文件
3. `.gitignore` 缺少对优化报告文件的排除规则

本次清理**不修改任何业务逻辑**，仅删除确认为死代码的文件和非源码的过程文件。

## 步骤 2：删除 2 个死代码兼容性包装文件

**删除**：
- `backend/src/modules/datasets/api/endpoints.py`
- `backend/src/modules/training/application/interfaces.py`

**依据**：
Python 模块解析规则：当 `endpoints.py` 文件和 `endpoints/` 包目录同时存在时，**Python 解析器优先使用包目录**（`endpoints/__init__.py`），完全忽略同名 `.py` 文件。这意味着这 2 个文件从未被 Python 加载过，属于死代码。

**已确认的导入链**：
- `router.py` 中 `from src.modules.datasets.api.endpoints import router` → 解析到 `endpoints/__init__.py`
- 14 处 `from src.modules.training.application.interfaces import ...` → 解析到 `interfaces/__init__.py`

**关键文件**（实际生效，不需修改）：
- `backend/src/modules/datasets/api/endpoints/__init__.py` - 新的 endpoints 包入口
- `backend/src/modules/training/application/interfaces/__init__.py` - 新的 interfaces 包入口

**验证**：
```bash
pytest tests/unit -v --tb=short
pytest tests/architecture -v --tb=short
python -c "from src.router import api_router; print('Routes:', len(api_router.routes))"
```

**风险**：SAFE
**回滚**：`git checkout HEAD -- <file>`
