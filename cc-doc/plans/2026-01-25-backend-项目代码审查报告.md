**问题**: `except Exception: pass` 吞掉所有异常，隐藏问题

except Exception:

## 🟠 HIGH 问题 (5个)

### H1. 默认 JWT 密钥泄露风险
**文件**: `src/shared/infrastructure/config.py:50`, `.env:2`
**问题**: JWT 密钥使用默认值 `"change-me-in-production"`
**风险**: 任何人获得代码库即可伪造 JWT Token
**建议**:
- 生产环境必须使用强随机密钥（32+ 字符）
- 使用 AWS Secrets Manager 管理敏感配置

### H2. 开发环境认证旁路
**文件**: `src/shared/api/middleware/auth.py:14-15, 89-104`
**问题**: 硬编码 `DEV_MOCK_TOKEN` 可获得 admin 权限
```python
DEV_MOCK_TOKEN = "dev-mock-token"  # 任何人可用此token获取admin权限
```
**风险**: 如环境变量误设为 development，生产环境将暴露
**建议**:
- 移除生产代码中的开发认证旁路
- 使用独立的开发配置文件

### H3. 数据库凭证明文存储
**文件**: `.env:3`
**问题**: `mysql+aiomysql://ai_training:ai_training_pass@localhost:3306/ai_training`
**建议**:
- 确保 `.env` 在 `.gitignore` 中
- 生产环境使用 AWS Secrets Manager

### H4. 大型函数需要拆分
**文件**: `src/modules/training/infrastructure/hyperpod/client.py:191-307`
**问题**: `_submit()` 函数 117 行，复杂度过高
**建议**: 拆分为多个私有方法:
- `_build_container_spec()`
- `_build_pod_spec()`
- `_apply_task_governance()`

### H5. 14个关键实体缺少单元测试
**缺失测试**:
| 模块 | 缺失实体 |
|------|---------|
| auth | User, LoginAttempt, PasswordHistory |
| quotas | ResourceQuota, ResourceLimitConfig |
| spaces | Space |
| audit | AuditLog |
| datasets | Dataset |
