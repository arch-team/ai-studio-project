## 1. 当前状态评估

### 1.1 已完成的改进 ✅

代码已经进行了重大重构，解决了之前计划中的主要问题：

#### 改进 1: 移除映射表，改用类属性

```python
# 旧方案（已废弃）
DOMAIN_EXCEPTION_MAP: dict[type[DomainError], int] = {
    EntityNotFoundError: 404,
    DuplicateEntityError: 409,
    # ...需要手动维护
}

# 新方案（当前）
class EntityNotFoundError(DomainError):
    http_status = 404
    error_code = "ENTITY_NOT_FOUND"
    # 异常处理器自动读取这些属性
```

**收益**：新增异常无需修改 exception_handlers.py，只需在异常类定义属性即可。

#### 改进 2: 统一响应格式

```json
{
  "error": {
    "code": "ENTITY_NOT_FOUND",
    "message": "TrainingJob with id '123' not found",
    "details": {
      "entity_type": "TrainingJob",
      "entity_id": "123"
    },
    "trace_id": "req-xxx"
  }
}
```

**设计参考**：简化版 RFC 9457 + Google Cloud API 风格

#### 改进 3: 结构化错误详情

```python
class EntityNotFoundError(DomainError):
    def get_details(self) -> dict[str, Any]:
        return {"entity_type": self.entity_type, "entity_id": self.entity_id}
```

**收益**：前端可以程序化处理错误详情。

### 1.2 仍存在的问题

#### 问题 1: Auth 与 Security 异常重复定义（🟡 中等）

**11 个异常**在两处重复定义：

| 异常类 | shared/security | auth/domain |
|--------|----------------|-------------|
| `UserNotFoundError` | ✓ | ✓ |
| `InvalidCredentialsError` | ✓ | ✓ |
| `AccountLockedError` | ✓ | ✓ |
| `TokenExpiredError` | ✓ | ✓ |
| `InvalidTokenError` | ✓ | ✓ |
| `PasswordExpiredError` | ✓ | ✓ |
| `PasswordTooWeakError` | ✓ | ✓ |
| `PasswordHistoryViolationError` | ✓ | ✓ |
| `InsufficientPermissionsError` | ✓ | ✓ |
| `SSOError` | ✓ | ✓ |
| `SSODegradedModeError` | ✓ | ✓ |

**当前设计意图**（根据文档注释）：
- `shared/infrastructure/security/exceptions.py` → 基础设施代码使用（JWT 中间件、Token 验证）
- `modules/auth/domain/exceptions.py` → 业务逻辑代码使用（AuthService、AccountService）

**问题**：
- 两个类名相同但实现略有不同（如构造函数签名）
- 容易混淆，不清楚该导入哪个
- 维护两套代码

#### 问题 2: 不完全符合 RFC 9457（🟢 可选改进）

当前响应格式缺少：
- `type`: 问题类型 URI
- `title`: 人类可读的简短标题
- `instance`: 问题发生位置
- `status`: HTTP 状态码（在响应体中）

**评估**：当前格式已经足够实用，RFC 9457 完全兼容是锦上添花。

## 2. 重新评估的方案

### 方案 A: 保持现状（推荐）

**理由**：
1. 当前设计已经很好，响应格式统一且实用
2. 两处异常定义有明确的使用场景划分
3. 重构风险大于收益

**行动**：
- 完善文档，明确两处异常的使用场景
- 考虑在 auth/domain/exceptions.py 改为重导出（减少代码重复）

### 方案 B: 合并异常定义（可选）

**改造**：将 auth/domain/exceptions.py 改为重导出 shared/security 异常

```python
# modules/auth/domain/exceptions.py (改造后)
"""Auth domain exceptions - 重导出以保持向后兼容."""

from src.shared.infrastructure.security.exceptions import (
    SecurityError,
    UserNotFoundError,
    InvalidCredentialsError,
    AccountLockedError,
    # ... 其他重导出
)

# Auth 模块独有异常
class AccountInactiveError(SecurityError):
    http_status = 401
    def __init__(self, message: str = "Account is not active"):
        super().__init__(message, code="ACCOUNT_INACTIVE")
```

**收益**：
- 减少代码重复
- 确保行为一致
- 保持向后兼容

**风险**：
- 需要统一两个模块的构造函数签名
- 可能影响现有业务代码

### 方案 C: 完善 RFC 9457 支持（可选）

**改造**：在响应中添加 RFC 9457 字段

**改造后的文件结构**：

```python
# modules/auth/domain/exceptions.py
"""Auth domain exceptions - 重导出 shared/security 异常以保持向后兼容.

使用指南:
--------
- auth 模块业务代码 → 使用本模块异常（实际为重导出）
- 基础设施代码 (middleware, jwt) → 使用 shared/infrastructure/security/exceptions.py
"""

from src.shared.infrastructure.security.exceptions import (
    SecurityError,
    AuthenticationError,
    UserNotFoundError,
    InvalidCredentialsError,
    AccountLockedError,
    AccountValidationError,
    TokenExpiredError,
    InvalidTokenError,
    PasswordExpiredError,
    PasswordTooWeakError,
    PasswordHistoryViolationError,
    InsufficientPermissionsError,
    SSOError,
    SSODegradedModeError,
)

# Auth 模块独有异常（不在 shared/security 中）
class AuthError(SecurityError):
    """Base exception for authentication errors."""
    http_status = 401
    def __init__(self, message: str, code: str = "AUTH_ERROR"):
        super().__init__(message, code=code)

class AccountInactiveError(AuthError):
    """Raised when account is not active."""
    http_status = 401
    def __init__(self, message: str = "Account is not active"):
        super().__init__(message, code="ACCOUNT_INACTIVE")

class TokenError(AuthError):
    """Base exception for token errors."""
    http_status = 401
    def __init__(self, message: str = "Token error", code: str = "TOKEN_ERROR"):
        super().__init__(message, code=code)

__all__ = [
    # 重导出的异常
    "SecurityError",
    "AuthenticationError",
    "UserNotFoundError",
    "InvalidCredentialsError",
    "AccountLockedError",
    "AccountValidationError",
    "TokenExpiredError",
    "InvalidTokenError",
    "PasswordExpiredError",
    "PasswordTooWeakError",
    "PasswordHistoryViolationError",
    "InsufficientPermissionsError",
    "SSOError",
    "SSODegradedModeError",
    # Auth 模块独有
    "AuthError",
    "AccountInactiveError",
    "TokenError",
]
```
