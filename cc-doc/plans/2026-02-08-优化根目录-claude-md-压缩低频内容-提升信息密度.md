# 优化根目录 CLAUDE.md — 压缩低频内容，提升信息密度

## Context

根目录 `CLAUDE.md`（148行）作为 Claude Code 的项目级上下文文件，**每次对话都会被自动加载**。经审计发现：
- Spec-Kit 章节（38行）描述的是低频规划操作，却占文件 25% 篇幅
- Architecture 章节（14行）信息密度低，且缺少前端架构
- TDD 测试分层表在各子项目 testing.md 中已有完整版本
- 英文章节标题与"所有文档使用中文"的项目要求矛盾
- Key Documentation 表与 `.claude/CLAUDE.md` 导航部分重叠

**目标**: 从 148 行压缩至 ~87 行（-41%），节约每次对话约 1500 token。

## 质量评估报告

### CLAUDE.md Quality Report

**Score: 72/100 (Grade: B)**

| 评估维度 | 得分 | 说明 |
|----------|------|------|
| 命令/工作流文档 | 12/20 | 无直接命令，仅有子项目引用指针 |
| 架构清晰度 | 14/20 | 后端有，前端缺失，CDK 仅引用 |
| 非显而易见模式 | 12/15 | 术语表、状态机、测试诚信原则有价值 |
| 简洁性 | 8/15 | Spec-Kit 占比过大，TDD 表与子项目重复 |
| 时效性 | 15/15 | 所有信息与项目实际状态一致 |
| 可操作性 | 11/15 | 多数是引用指针，直接可操作内容偏少 |

### 问题清单

1. **Spec-Kit 章节过重** (38行 / 25%): 低频规划操作占用高频加载位置
2. **架构信息不完整**: 缺少前端架构（Feature-Sliced Design + Cloudscape）
3. **英文标题不一致**: 与"所有文档使用中文"矛盾
4. **TDD 分层表冗余**: 各子项目 testing.md 已有完整版
5. **文档导航表重叠**: 与 `.claude/CLAUDE.md` 部分重复

## 优化操作清单

### 操作 1: Spec-Kit 章节迁移至 common.md（-35行）

**涉及文件**: `CLAUDE.md` 第110-147行, `.claude/rules/common.md`

**操作**:
- 将根 CLAUDE.md 中 Spec-Kit 完整内容（目录结构、文件职责速查表、工作流程、一致性检查）迁移到 `.claude/rules/common.md` 末尾
- 根 CLAUDE.md 替换为 3 行引用：

```markdown
## Spec-Kit 规范驱动开发

本项目使用 Spec-Kit 进行规范驱动开发。详细文件体系和命令参见 `.claude/rules/common.md`。
```

**理由**: Spec-Kit 操作仅在规划阶段使用，common.md 定位是"跨项目通用规则"，适合承载此内容。`/speckit.*` 命令已在 `.claude/commands/` 中定义，Claude Code 可自动发现。

### 操作 2: TDD 描述精简（-17行）

**涉及文件**: `CLAUDE.md` 第48-78行

**操作**:
- 保留 TDD 核心循环（压缩为单行）和测试诚信原则
- 删除测试分层策略表（7行，各子项目 testing.md 已有完整版）
- 删除子项目测试命令引用列表（4行，改为通用引用）
- SDK-First 引用保持简洁

**替换为**:
```markdown
## 核心开发原则

### TDD 工作流

本项目全面采用测试驱动开发 (TDD)。

**核心循环**: Red (先写失败测试) → Green (最少代码通过) → Refactor (重构保持通过)

**测试诚信原则**: 切勿为让测试通过而伪造结果。测试失败 = 代码有问题，必须修复代码。

> 各模块测试命令和分层策略参见对应子项目 CLAUDE.md

### SDK-First 原则

> 详细 SDK 使用规范请参见 `backend/CLAUDE.md`
```
